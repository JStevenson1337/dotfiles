<h2><a id="intro" class="anchor" href="#intro">¶</a>Intro</h2><div class="outline-text-2" id="text-orgb768cf7">
<p>
One of Org Mode&rsquo;s most useful features is its publishing system which allows you to turn Org files into many other formats.
</p>

<p>
In this video, I&rsquo;ll show you how to use it to generate a simple but nice looking website using Org Mode&rsquo;s HTML publishing capabilities!
</p>

<div class="cta center">
If you find this guide helpful, please consider supporting System Crafters via the links on the <a href="/how-to-help/#support-my-work">How to Help</a> page!
</div>

<h2><a id="getting-started" class="anchor" href="#getting-started">¶</a>Getting started</h2><div class="outline-text-2" id="text-org2661964">
<p>
To get started, we&rsquo;ll need a place to put all of the code we&rsquo;ll use to generate the site along with the <code>.org</code> files that will provide the site&rsquo;s content.
</p>

<p>
We&rsquo;ll create a folder called <code>my-org-site</code> with a subfolder called <code>content</code> which contains a single <code>index.org</code> file. This will create a file called <code>index.html</code> which will be the root page of our site!
</p>

<p>
The <code>.org</code> file you place in there can be very simple like the one I&rsquo;ll show you. Don&rsquo;t put any code blocks in there yet, we&rsquo;ll cover that a bit later in this video.
</p>

<p>
Most of what we&rsquo;ll do in this episode only requires Emacs version 24.5 or higher with the Org Mode package that comes built-in.
</p>
</div>

<h2><a id="creating-the-build-script" class="anchor" href="#creating-the-build-script">¶</a>Creating the build script</h2><div class="outline-text-2" id="text-org06f8cba">
<p>
Now we&rsquo;re going to create a script in Emacs Lisp that will generate the HTML files for your Org documents by running a single shell command.
</p>

<p>
Let&rsquo;s call it <code>build-site.el</code>:
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Load the publishing system</span>
(<span class="org-keyword">require</span> '<span class="org-constant">ox-publish</span>)

(message <span class="org-string">"Build complete!"</span>)</pre>

<p>
We also need a shell script that can run our Emacs Lisp script. We&rsquo;ll call that <code>build.sh</code>:
</p>

<pre><span class="org-comment-delimiter">#</span><span class="org-comment">!/bin/sh</span>
emacs -Q --script build-site.el</pre>

<p>
Don&rsquo;t forget to use <code>chmod +x build.sh</code> to make the script executable!
</p>

<p>
<b>NOTE:</b> If you&rsquo;re on Windows, you can create a <code>build.cmd</code> file that invokes Emacs the same way.
</p>

<p>
The <code>-Q</code> parameter prevents Emacs from loading your own Emacs configuration which will help ensure that this same script can be executed cleanly on other machines once we get ready to start publishing this site.
</p>
</div>

<h2><a id="setting-up-the-publish-project" class="anchor" href="#setting-up-the-publish-project">¶</a>Setting up the publish project</h2><div class="outline-text-2" id="text-orgf23373f">
<p>
Now that we have our basic script ready, we can set up a Org publishing configuration for our website.
</p>

<p>
We&rsquo;ll add the following snippet into our <code>build-site.el</code> file:
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Define the publishing project</span>
(<span class="org-keyword">setq</span> org-publish-project-alist
(list
(list <span class="org-string">"my-org-site"</span>
<span class="org-builtin">:recursive</span> t
<span class="org-builtin">:base-directory</span> <span class="org-string">"./content"</span>
<span class="org-builtin">:publishing-directory</span> <span class="org-string">"./public"</span>
<span class="org-builtin">:publishing-function</span> 'org-html-publish-to-html)))

<span class="org-comment-delimiter">;; </span><span class="org-comment">Generate the site output</span>
(org-publish-all t)</pre>

<p>
This configures the <code>org-publish-project-alist</code> variable to define a publishing project for our site. Read the documentation of this variable using <code>M-x describe-variable</code> for a lot more information!
</p>

<p>
We use the name <code>my-org-site</code> but you can call the project whatever you want!
</p>

<ul class="org-ul">
<li><code>:base-directory</code> tells Org the folder of files that we want to publish to HTML</li>
<li><code>:publishing-directory</code> configures the output folder for all published files</li>
<li><code>:recursive</code> tells Org to publish <code>.org</code> files under all subdirectories of <code>:base-directory</code></li>
<li><code>:publishing-function</code> tells Org that we want to publish the files as HTML</li>
</ul>

<p>
The <code>org-publish-all</code> function at the end tells Org to publish every project configured in <code>org-publish-project-alist</code>! The <code>t</code> parameter tells it to regenerate all files regardless of when they were last generated.
</p>

<p>
We can now generate the output of our site by running <code>build.sh</code>! Check out the <code>public/</code> folder after running it.
</p>
</div>

<h2><a id="previewing-the-generated-site" class="anchor" href="#previewing-the-generated-site">¶</a>Previewing the generated site</h2><div class="outline-text-2" id="text-orgad29e8c">
<p>
Let&rsquo;s take a look at the basic site we just generated. There&rsquo;s a great Emacs package called <a href="https://github.com/skeeto/emacs-web-server">simple-httpd</a> which can host your files as a website on your local machine so that you can pull it up in your browser.
</p>

<p>
You can install <code>simple-httpd</code> from <a href="https://melpa.org/#/getting-started">MELPA</a> using <code>M-x package-install</code> or by putting the following snippet <b>in your Emacs configuration</b> if you have <code>use-package</code> installed and MELPA configured as a package source:
</p>

<pre>(<span class="org-keyword">use-package</span> <span class="org-constant">simple-httpd</span>
<span class="org-builtin">:ensure</span> t)</pre>

<p>
Now you can run <code>M-x httpd-serve-directory</code>. It will prompt you for a directory to serve from within Emacs.
</p>

<p>
Select the path of your <code>public/</code> directory and then open your browser to <a href="http://localhost:8080">http://localhost:8080</a> to see the preview of your site! Set <code>httpd-port</code> to change the default port if necessary.
</p>

<p>
Any time you regenerate the site files, you can just reload the page to see the result of changes you made!
</p>
</div>

<h2><a id="improving-the-html-output" class="anchor" href="#improving-the-html-output">¶</a>Improving the HTML output</h2><div class="outline-text-2" id="text-org3df3c3d">
<p>
There are a few things about the page we just looked at that I&rsquo;d like to change to make the output look a bit cleaner:
</p>

<ul class="org-ul">
<li>Section numbers</li>
<li>Author name</li>
<li>Publish timestamp</li>
<li>Validate link</li>
</ul>

<p>
Luckily the publish system is very customizable so I&rsquo;ll show you how we can disable a few things in the project configuration to get a better result.
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Define the publishing project</span>
(<span class="org-keyword">setq</span> org-publish-project-alist
(list
(list <span class="org-string">"org-site:main"</span>
<span class="org-builtin">:recursive</span> t
<span class="org-builtin">:base-directory</span> <span class="org-string">"./content"</span>
<span class="org-builtin">:publishing-function</span> 'org-html-publish-to-html
<span class="org-builtin">:publishing-directory</span> <span class="org-string">"./public"</span>
<span class="org-builtin">:with-author</span> nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Don't include author name</span>
<span class="org-builtin">:with-creator</span> t <span class="org-comment-delimiter">;; </span><span class="org-comment">Include Emacs and Org versions in footer</span>
<span class="org-builtin">:with-toc</span> t <span class="org-comment-delimiter">;; </span><span class="org-comment">Include a table of contents</span>
<span class="org-builtin">:section-numbers</span> nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Don't include section numbers</span>
<span class="org-builtin">:time-stamp-file</span> nil))) <span class="org-comment-delimiter">;; </span><span class="org-comment">Don't include time stamp in file</span></pre>

<p>
We&rsquo;ll need to set a different variable to get rid of the validate link at the bottom:
</p>

<pre>(<span class="org-keyword">setq</span> org-html-validation-link nil)</pre>
</div>

<h2><a id="improving-the-page-styling" class="anchor" href="#improving-the-page-styling">¶</a>Improving the page styling</h2><div class="outline-text-2" id="text-org448339e">
<p>
At this point have a decent basic output for our website, but what if we want to make it look a little bit nicer?
</p>

<p>
By setting a few more variables, we can use a nice stylesheet to give our site a more polished look:
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Customize the HTML output</span>
(<span class="org-keyword">setq</span> org-html-validation-link nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Don't show validation link</span>
org-html-head-include-scripts nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Use our own scripts</span>
org-html-head-include-default-style nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Use our own styles</span>
org-html-head <span class="org-string">"&lt;link rel=\"stylesheet\" href=\"https://cdn.simplecss.org/simple.min.css\" /&gt;"</span>)</pre>

<p>
This will remove the default JavaScript and CSS code that gets injected into the HTML output by default and replace it with the link to a nice stylesheet called <a href="https://simplecss.org/">Simple.css</a> (or a CSS file of your own!).
</p>

<p>
Let&rsquo;s regenerate the site and take a look!
</p>
</div>

<h2><a id="generating-pages-with-code-blocks" class="anchor" href="#generating-pages-with-code-blocks">¶</a>Generating pages with code blocks</h2><div class="outline-text-2" id="text-org974497b">
<p>
So far we&rsquo;ve been looking at a very simple example page that doesn&rsquo;t really have much on it. What happens when we try to generate a more elaborate Org file containing code blocks?
</p>

<p>
Let&rsquo;s try it out with another file, a version of my literate Emacs configuration called <code>Emacs.org</code>!
</p>

<p>
If you&rsquo;re generating a site that features code blocks like a coding blog or a literate Emacs configuration, you&rsquo;ll probably see an error like this when you generate the site:
</p>

<pre>Cannot fontify source block (htmlize.el &gt;= 1.34 required)</pre>

<p>
To resolve this issue, you will need to install the <code>htmlize</code> package from MELPA. We can automate the process of installing this package by adding the following snippet to our <code>build-site.el</code> file:
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Set the package installation directory so that packages aren't stored in the</span>
<span class="org-comment-delimiter">;; </span><span class="org-comment">~/.emacs.d/elpa path.</span>
(<span class="org-keyword">require</span> '<span class="org-constant">package</span>)
(<span class="org-keyword">setq</span> package-user-dir (expand-file-name <span class="org-string">"./.packages"</span>))
(<span class="org-keyword">setq</span> package-archives '((<span class="org-string">"melpa"</span> . <span class="org-string">"https://melpa.org/packages/"</span>)
(<span class="org-string">"elpa"</span> . <span class="org-string">"https://elpa.gnu.org/packages/"</span>)))

<span class="org-comment-delimiter">;; </span><span class="org-comment">Initialize the package system</span>
(package-initialize)
(<span class="org-keyword">unless</span> package-archive-contents
(package-refresh-contents))

<span class="org-comment-delimiter">;; </span><span class="org-comment">Install dependencies</span>
(package-install 'htmlize)</pre>

<p>
In this snippet, we load Emacs&rsquo; package manager and configure the <code>package-user-dir</code> to be a sub-directory of our project folder. This allows you to install packages for the script without mixing them up with the packages of your personal Emacs configuration!
</p>

<p>
The next thing we do is add MELPA to the package archive list and refresh the package archive so that <code>htmlize</code> can be found. Finally, we call <code>package-install</code> to install it!
</p>

<p>
In a future video I&rsquo;ll show you how to convert your Emacs color theme to a CSS file that you can use to colorize source blocks with exactly the same colors used in the theme!
</p>
</div>

<h2><a id="linking-between-pages" class="anchor" href="#linking-between-pages">¶</a>Linking between pages</h2><div class="outline-text-2" id="text-orgec99f5a">
<p>
One last thing I want to show you is how you can create links between pages on your site. Let&rsquo;s open up <code>index.org</code> and create a link to the <code>Emacs.org</code> file.
</p>

<p>
Inside <code>index.org</code>, we can press <code>C-c C-l</code> (<code>org-insert-link</code>), enter the path of the Org file we want to link to (<code>./Emacs.org</code>), press Enter, then enter the text for the link (&ldquo;My Emacs configuration&rdquo;).
</p>

<p>
When you generate the site again, you should be able to follow the link between pages because Org&rsquo;s publishing system will convert the link to the proper output extension.
</p>

<p>
It will also complain when a linked file can&rsquo;t be found!
</p>

<pre>Debugger entered--Lisp error: (user-error <span class="org-string">"Unable to resolve link: \"Emacs2.org\""</span>)</pre>
</div>

<h2><a id="the-final-build-script" class="anchor" href="#the-final-build-script">¶</a>The final build script</h2><div class="outline-text-2" id="text-orgaf8e88e">
<p>
I&rsquo;ve committed all of the code you see in this video to the following GitHub repository:
</p>

<p>
<a href="https://github.com/SystemCrafters/org-website-example/">https://github.com/SystemCrafters/org-website-example/</a> (see the <a href="https://github.com/SystemCrafters/org-website-example/commit/1ee251e97f5b4d6c614936030203cd7368d4adc8">commit for this episode</a>)
</p>

<p>
In the next video I&rsquo;ll show you how to automatically publish an Org-based website to Git hosting services like GitHub Pages and Sourcehut Pages!
</p>

<p>
Here is the final form of our <code>build-site.el</code> script:
</p>

<pre><span class="org-comment-delimiter">;; </span><span class="org-comment">Set the package installation directory so that packages aren't stored in the</span>
<span class="org-comment-delimiter">;; </span><span class="org-comment">~/.emacs.d/elpa path.</span>
(<span class="org-keyword">require</span> '<span class="org-constant">package</span>)
(<span class="org-keyword">setq</span> package-user-dir (expand-file-name <span class="org-string">"./.packages"</span>))
(<span class="org-keyword">setq</span> package-archives '((<span class="org-string">"melpa"</span> . <span class="org-string">"https://melpa.org/packages/"</span>)
(<span class="org-string">"elpa"</span> . <span class="org-string">"https://elpa.gnu.org/packages/"</span>)))

<span class="org-comment-delimiter">;; </span><span class="org-comment">Initialize the package system</span>
(package-initialize)
(<span class="org-keyword">unless</span> package-archive-contents
(package-refresh-contents))

<span class="org-comment-delimiter">;; </span><span class="org-comment">Install dependencies</span>
(package-install 'htmlize)

<span class="org-comment-delimiter">;; </span><span class="org-comment">Load the publishing system</span>
(<span class="org-keyword">require</span> '<span class="org-constant">ox-publish</span>)

<span class="org-comment-delimiter">;; </span><span class="org-comment">Customize the HTML output</span>
(<span class="org-keyword">setq</span> org-html-validation-link nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Don't show validation link</span>
org-html-head-include-scripts nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Use our own scripts</span>
org-html-head-include-default-style nil <span class="org-comment-delimiter">;; </span><span class="org-comment">Use our own styles</span>
org-html-head <span class="org-string">"&lt;link rel=\"stylesheet\" href=\"https://cdn.simplecss.org/simple.min.css\" /&gt;"</span>)
